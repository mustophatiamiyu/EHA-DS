# Icons

The EHA Design System uses **Google Material Icons** as its primary icon library. This provides a comprehensive set of over 2,000+ industry-standard icons that are well-documented and widely recognized.

## Primary Library: Google Material Icons

Material Icons follow the Material Design guidelines and are available as high-quality SVG React components via the `@mui/icons-material` package.

### Why Material Icons?

- **Consistency**: Used across countless applications, making them familiar to users.
- **Variety**: Over 2,000 icons in multiple styles (Filled, Outlined, Rounded, etc.).
- **Performance**: Supports tree-shaking; only the icons you use are bundled.
- **Ease of Use**: Imported directly as React components.

### Usage in EHA-DS

Icons can be passed to various components such as `Button`, `MetricCard`, and `Sidenav`. They automatically inherit styles when used within these components, but can also be customized individually.

#### 1. Basic Import

```tsx
import HomeIcon from '@mui/icons-material/Home';
import SettingsIcon from '@mui/icons-material/Settings';
import AssessmentIcon from '@mui/icons-material/Assessment';
```

#### 2. Usage in Components

**MetricCard Example:**

```tsx
<MetricCard 
    value="2,140" 
    label="Active Sessions" 
    icon={<AssessmentIcon />} 
/>
```

**Sidenav Example:**

```tsx
const items = [
    { eventKey: '1', title: 'Dashboard', icon: <HomeIcon /> },
    { eventKey: '2', title: 'Settings', icon: <SettingsIcon /> },
];

<Sidenav items={items} />
```

### Styling Icons

While EHA-DS components handle most styling, you can manually style icons using standard React props:

- **color**: Use `primary`, `secondary`, `action`, `error`, `disabled`, or `inherit`.
- **fontSize**: Use `small`, `medium` (default), `large`, or a custom value via `sx` or `style`.
- **style / sx**: For custom CSS properties like `color: designTokens.colors.primary50`.

```tsx
<HomeIcon color="primary" fontSize="large" />
<SettingsIcon style={{ color: '#0052CC' }} />
```
